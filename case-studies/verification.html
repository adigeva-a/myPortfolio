Background

A signal hiding in the funnel

While running routine funnel analysis on the registration flow, I spotted something unusual: the phone verification step had a dramatically higher drop-off rate than any comparable step in the funnel. This wasn't a gradual decline ‚Äî it was a sharp cliff that had gone unaddressed.

What made it initially hard to diagnose was that the drop-off manifested the same way whether users were experiencing an SMS delivery failure, a confusing UI, or a timeout. The symptom was one thing; the root causes turned out to be three distinct, compounding problems.

Root Cause Analysis

Three distinct failure modes

By cross-referencing Mixpanel event data with SMS delivery logs and support tickets, I isolated three independent root causes ‚Äî each requiring a different type of fix.

Failure Mode 01
Infrastructure: SMS non-delivery
SMS delivery rates below 60% in 3 key markets (SE Asia, LATAM sub-regions). Root cause: SMS routing through a provider with poor carrier relationships in these regions.
Impact: ~40% of drop-offs in affected markets
Failure Mode 02
UX: No recovery path visible
Users who didn't receive a code had no obvious next step. "Resend" link existed but was below the fold and styled as secondary text. Rage clicks on the input field were the most common behavior before abandonment.
Impact: ~35% of drop-offs across all markets
Failure Mode 03
Product: Timeout too short
60-second OTP expiry created race conditions where valid codes expired before slow SMS delivery completed. Particularly acute on mobile networks with higher latency.
Impact: ~25% of drop-offs, concentrated on mobile
Research Conducted

How I connected the dots

The investigation required connecting data from sources that had never been correlated before: Mixpanel funnel events, SMS gateway delivery logs from our infrastructure provider, FullStory session recordings, and support ticket categorizations.

The breakthrough came when I matched user IDs from Mixpanel abandonment events with SMS delivery failure logs ‚Äî and found that in 3 countries, delivery failure preceded abandonment in over 70% of drop-off cases.

This case is notable because the root cause spanned three domains ‚Äî SMS routing, UX affordances, and OTP configuration ‚Äî and required cross-functional coordination across Engineering, Infrastructure, Design, and Legal to resolve fully. Fixing any one in isolation would have recovered roughly a third of the drop-off.
Key Learnings

What this built in me

  • Single-channel flows are fragile at global scale ‚Äî always design channel fallbacks for critical steps
  • Infrastructure failures surface as UX problems in the data. Cross-functional root cause analysis is essential to see the full picture
  • Specific, actionable error messages dramatically improve recovery. "Verification failed" tells users nothing. "Try voice verification instead" gives them a path forward.
  • Support tickets are a leading indicator for funnel problems ‚Äî mining them early would have caught this months sooner
  • Segmenting by market and device type before diagnosing saves weeks. The problem looked uniform from the top; it was highly concentrated when sliced properly
  • Sometimes the most impactful PM work isn't a new feature ‚Äî it's fixing what's quietly broken
